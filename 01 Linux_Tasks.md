# 🐧 Linux DevOps Lab Tasks 

###  Task 1: System Exploration and Navigation
**Objective**  
Explore the Linux system environment and navigate directories like a system user.

**Requirements**
- Identify current working directory  
- List all files including hidden files in the current directory  
- Navigate to /tmp and review its contents  
- Return back to the home directory  
- Identify the current logged-in user  
- Display system information  

---

###  Task 2: Project Workspace Initialization
**Objective**  
Set up a structured workspace for a DevOps project.

**Requirements**
- Create a main directory named linux_lab  
- Inside it create a subdirectory day1  
- Create files a.txt, b.txt, c.txt inside day1  
- Create a nested directory structure project/app/logs  
- Display the full directory structure  

---

###  Task 3: Application Data Preparation
**Objective**  
Create and manage application data files for processing.

**Requirements**
- Add sample application data into a.txt  
- Append additional log entries  
- Display file content using multiple methods  
- Modify content using a text editor  
- Verify updated content  

---

###  Task 4: Backup and File Relocation
**Objective**  
Simulate backup and file organization in a system.

**Requirements**
- Create a backup copy of a.txt as backup.txt  
- Move backup.txt to /tmp for temporary storage  
- Rename b.txt to new.txt  
- Verify file locations after operations  

---

###  Task 5: System Cleanup Operation
**Objective**  
Perform cleanup of unnecessary files and directories.

**Requirements**
- Remove new.txt from the workspace  
- Delete the entire project directory recursively  
- Confirm that files and directories are removed successfully  

---

###  Task 6: Securing Sensitive Files
**Objective**  
Apply strict permission controls to sensitive data.

**Requirements**
- Create a file named secure.txt  
- Restrict access so only the owner can read and write  
- Temporarily grant full access and then revert back to secure settings  
- Verify permission changes at each step  

---

###  Task 7: Ownership Assignment for Team Access
**Objective**  
Assign file ownership for controlled access.

**Requirements**
- Create a user named devops  
- Assign ownership of secure.txt to devops  
- Verify ownership details of the file  

---

###  Task 8: File Discovery and Content Inspection
**Objective**  
Locate files and extract meaningful data.

**Requirements**
- Search for all .txt files within the home directory  
- Identify files containing the keyword Linux  
- Display matching lines from the results  

---

###  Task 9: Application Log Monitoring
**Objective**  
Monitor and analyze application logs.

**Requirements**
- Create a file data.txt with multiple log entries  
- Display the first few log entries  
- Display the most recent entries  
- Continuously monitor the file for updates  
- Append new log entries and observe changes  

---

###  Task 10: Data Sorting and Organization
**Objective**  
Organize and structure raw data.

**Requirements**
- Create a file with unsorted entries  
- Sort the data alphabetically  
- Sort the data in reverse order  
- Save the sorted output into a new file  
- Verify correctness of sorted data  

---

###  Task 11: Error Log Extraction and Analysis
**Objective**  
Extract and analyze error information from logs.

**Requirements**
- Create a log file containing mixed entries (INFO, ERROR, WARNING)  
- Extract only lines containing error  
- Perform case-insensitive search  
- Display line numbers of matching entries  
- Count total number of error occurrences  

---

###  Task 12: File Metrics and Reporting
**Objective**  
Generate detailed statistics for files.

**Requirements**
- Create a file with multiple lines and words  
- Count total number of lines  
- Count total number of words  
- Count total number of characters  
- Display all statistics in a single output  

---

###  Task 13: Script Execution Permission Setup
**Objective**  
Prepare script files with proper execution permissions.

**Requirements**
- Create a script file script.sh  
- Grant execute permission only to the owner  
- Remove unnecessary permissions from group and others  
- Verify permission configuration
